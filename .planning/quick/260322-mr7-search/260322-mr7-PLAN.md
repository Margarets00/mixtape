### 2. src/components/SearchTab.tsx 수정

`SearchTabProps` 에 두 prop 추가:
```typescript
interface SearchTabProps {
  dispatch: React.Dispatch<QueueAction>;
  queue: QueueItem[];
  onPreview: (track: PreviewTrack) => void;
  onNavigateSettings: () => void;
  downloadedIds?: Set<string>;
  searchState: SearchState;                        // 추가
  onSearchStateChange: (s: SearchState) => void;   // 추가
}
```

내부 `useState` 6개(query, results, isSearching, hasSearched, usedFallback, isPlaylist, playlistTracks, selectedIds, playlistLoading) 를 모두 제거한다.
대신 `const { query, results, ... } = searchState;` 로 구조분해하고,
각 setter(`setQuery` 등) 를 `onSearchStateChange({ ...searchState, query: newVal })` 형태의 헬퍼로 교체한다.

변경이 많으므로 상단에 헬퍼를 만들면 편하다:
```typescript
const update = (patch: Partial<SearchState>) =>
  onSearchStateChange({ ...searchState, ...patch });
```

이후 기존 `setQuery(v)` → `update({ query: v })`, `setResults(r)` → `update({ results: r })` 등으로 치환.

`setPlaylistTracks((prev) => [...prev, track])` 처럼 함수형 업데이트를 쓰는 부분은
콜백 패턴이 필요하므로 `onSearchStateChange` 를 직접 호출하는 방식으로:
```typescript
onSearchStateChange({
  ...searchState,
  playlistTracks: [...searchState.playlistTracks, track],
});
```

단, `handleSearch` 같은 async 함수 내에서 searchState 가 클로저로 캡처되는 문제를 피하려면
`onTrack.onmessage` 콜백을 `setPlaylistTracks` 대신 `onSearchStateChange` 를 직접 쓰는 게 아니라
React `useState` 함수형 업데이터와 동일하게 동작하는 ref 패턴을 쓴다:

```typescript
// SearchTab 내부에 ref 를 하나 유지해 최신 상태를 클로저 없이 참조
const searchStateRef = useRef(searchState);
useEffect(() => { searchStateRef.current = searchState; }, [searchState]);

// onTrack.onmessage 내부:
onTrack.onmessage = (event) => {
  if (event.type === 'Track' && event.data) {
    const track: PlaylistTrack = { ... };
    onSearchStateChange({
      ...searchStateRef.current,
      playlistTracks: [...searchStateRef.current.playlistTracks, track],
    });
  } else if (event.type === 'Done') {
    onSearchStateChange({ ...searchStateRef.current, playlistLoading: false });
  } else if (event.type === 'Error') {
    onSearchStateChange({ ...searchStateRef.current, playlistLoading: false });
  }
};
```

`SearchState` 타입은 App.tsx 에서 export 하고 SearchTab.tsx 에서 import 한다.
마찬가지로 `SearchResult` 타입은 SearchResultRow 에서, `PlaylistTrack` 타입은 PlaylistTrackRow 에서 이미 import 하고 있으므로 그대로 유지.
