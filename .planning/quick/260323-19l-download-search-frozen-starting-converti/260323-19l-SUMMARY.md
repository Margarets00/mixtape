# Quick Task 260323-19l: Download/Search Frozen UI Gaps Summary

**One-liner:** Added DownloadEvent::Starting with shimmer UI, converting elapsed timer (Xs), and search elapsed timer to eliminate frozen-looking UI gaps in download and search flows.

## Objective

Fix three UI moments that made the app look frozen: (1) download pending-to-downloading gap (2-5s title fetch with no feedback), (2) converting state with no time feedback, (3) search initial gap with no elapsed indicator.

## Tasks Completed

| # | Task | Commit | Files |
|---|------|--------|-------|
| 1 | Add Starting event to Rust backend + wire frontend status type | d83fc3b | download.rs, queue.rs, App.tsx, QueueTab.tsx |
| 2 | QueueItem starting state UI + converting elapsed counter | 69fe9c8 | QueueItem.tsx |
| 3 | Search elapsed time counter | 28126dd | SearchTab.tsx |

## What Was Built

**Task 1 — Starting event pipeline:**
- Added `DownloadEvent::Starting` unit variant to `download.rs` enum
- Emits `Starting` in `download()` before the `is_playlist` branch (covers title-fetch gap)
- Emits `Starting` in `queue_download()` after the 2s delay, before title fetch
- Added `{ type: 'starting' }` to `QueueItemStatus` union in `App.tsx`
- Added `'Starting'` to `DownloadEvent` type union in `QueueTab.tsx` and dispatches `UPDATE_STATUS` with `status: { type: 'starting' }`

**Task 2 — Starting state UI + converting elapsed:**
- Added `useEffect` import to `QueueItem.tsx`
- `starting` case in `getBackground()` returns blue tint `rgba(183, 213, 255, 0.2)`
- Shimmer `@keyframes` injected via `<style>` element (shimmer is not in global.css)
- Full-width shimmer background fill div for `starting` state
- Starting text block shows title + animated shimmer "starting..." label
- `elapsedSecs` state + `useEffect` tied to `status.type === 'converting'`, resets on any other state
- Converting renders `CONVERTING... (Xs)` with incrementing counter
- Cancel button condition includes `starting` state

**Task 3 — Search elapsed counter:**
- `searchElapsed` state + `useEffect` tied to `isSearching` in `SearchTab`
- Counter starts at 0, increments per second while searching, resets when `isSearching` goes false
- Searching text shows `~ searching... ~` first second, then `~ searching... (1s) ~` etc.
