## What Changed

### Task 1: Cookie Temp File Caching

**Problem:** `cookie_browser_args()` was called on every preview/download/queue invocation, causing macOS Keychain to prompt repeatedly.

**Solution:**
- Added `cookie_file_path: Mutex<Option<String>>` to `AppState`
- Added `extract_cookies_to_tempfile(app)` — runs `yt-dlp --cookies-from-browser {browser} --cookies /tmp/yt-cookies-{pid}.txt --skip-download --no-warnings https://www.youtube.com` once at startup, stores path in state
- Replaced `cookie_browser_args()` with `cookie_file_args()` — reads cached file path, returns `["--cookies", "/tmp/yt-cookies-{pid}.txt"]`
- `lib.rs` setup: chains `detect_cookie_browser` → `extract_cookies_to_tempfile` in a single spawned async block
- `lib.rs` `RunEvent::Exit`: deletes the temp file on app exit
- All callers (download.rs, preview.rs, queue.rs) updated to use `cookie_file_args()`

### Task 2: URL Search Bug Fixes

**Bug 1 — bare URL not recognized:** `youtube.com/watch?v=abc` was treated as a keyword search. Fixed by normalizing bare `youtube.com` / `youtu.be` URLs with `https://` prefix before the `is_url` check.

**Bug 2 — URL search missing cookies:** URL branch of `search()` did not inject cookie args. Fixed by adding `app: tauri::AppHandle` parameter and calling `cookie_file_args()` for the yt-dlp URL lookup. Also added `use tauri::Manager` import required for `app.state()`.

**Bug 3 — Radio Mix URL treated as single video:** `isPlaylistUrl()` only matched `/playlist?list=`. Fixed by adding `url.includes('list=RD')` check to also match YouTube Radio Mix / Auto-generated playlists.
