**Step 3: Refactor `queue_download` in `src-tauri/src/queue.rs` to wrap the download-and-wait logic in a retry loop:**

The current code spawns yt-dlp once and waits. Restructure so that the inner spawned task (lines 105-134, the one holding `_permit`) runs a retry loop:

```rust
const BACKOFF_SECS: [u64; 3] = [30, 60, 120];
```

Inside the spawned task (where `_permit` is held), replace the simple `child.wait()` match with a loop:

1. Keep the yt-dlp spawn and stdout/stderr streaming INSIDE the loop (each retry needs a fresh process).
2. For stderr, collect lines into a `Vec<String>` (in addition to sending events) so we can inspect them after the process exits.
3. After `child.wait()`:
   - If `status.success()` => send `Done`, break.
   - If not success, check if ANY collected stderr line satisfies `is_rate_limit_error(line)`:
     - If yes AND `attempt < 3`:
       - Send `RetryWait { attempt: attempt + 1, wait_secs: BACKOFF_SECS[attempt], remaining_secs: BACKOFF_SECS[attempt] }` event.
       - Run a countdown loop: for each second of the backoff duration, sleep 1 second and send `RetryWait { attempt: attempt + 1, wait_secs: BACKOFF_SECS[attempt], remaining_secs: remaining }` so the frontend can update the countdown.
       - Increment `attempt`, continue the loop (re-spawn yt-dlp).
     - If yes AND `attempt >= 3`: send `Error { message: "YouTube rate limit (429). Tried 3 times with backoff. Try again later." }`, break.
     - If not a 429 error: send the original error message, break.
   - On `Err(e)` from wait: send Error, break.

**Important implementation details:**
- The `_permit` (semaphore slot) must remain held across all retry attempts — do NOT release it between retries.
- PID registration must be updated for each new child process spawn within the retry loop. Remove old PID, insert new PID.
- The `on_event` channel must be cloned appropriately for each iteration's stdout/stderr readers.
- Cancel support must still work during backoff sleep: check if `item_id` was removed from `queue_pids` during the countdown loop, and if so, break out.
- For stderr collection during retries: use an `Arc<Mutex<Vec<String>>>` that the stderr reader task pushes lines into, and the main task reads after process exit.
- The title fetch (`--print title`) happens ONCE before the loop (keep it outside). Only the yt-dlp download process is retried.
