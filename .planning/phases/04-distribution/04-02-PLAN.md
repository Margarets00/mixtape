<objective>
Create the GitHub Actions release workflow that produces signed, notarized artifacts for all three platforms on every release tag.

Purpose: This is the final piece that makes the app distributable. When a `v*` tag is pushed, three CI jobs build macOS universal .dmg (signed + notarized), Windows NSIS .exe (EV signed), and Linux AppImage. The tauri-action automatically generates `latest.json` for the in-app updater wired in Plan 01.

Output: `.github/workflows/release.yml` with three jobs and a human verification checkpoint.
</objective>

  <action>
    Create `.github/workflows/release.yml` per D-15, D-16, D-17. The workflow has three separate named jobs (not a matrix), all must succeed.

    **Workflow trigger (per D-15):**
    ```yaml
    name: Release
    on:
      push:
        tags: ['v*']
    ```

    **Job 1: build-macos (per D-08, D-09, D-11)**
    - `runs-on: macos-latest`
    - Steps:
      1. `actions/checkout@v4`
      2. `actions/setup-node@v4` with `node-version: lts/*` and `cache: npm`
      3. `dtolnay/rust-toolchain@stable` with `targets: aarch64-apple-darwin,x86_64-apple-darwin`
      4. `swatinem/rust-cache@v2`
      5. `npm ci`
      6. `bash scripts/download-sidecars-ci.sh universal-apple-darwin` (name: "Download sidecars (macOS universal)")
      7. `tauri-apps/tauri-action@v0` with:
         - env: `GITHUB_TOKEN`, `APPLE_CERTIFICATE`, `APPLE_CERTIFICATE_PASSWORD`, `APPLE_SIGNING_IDENTITY`, `APPLE_ID`, `APPLE_PASSWORD`, `APPLE_TEAM_ID`, `TAURI_SIGNING_PRIVATE_KEY`, `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` (all from `secrets`)
         - with: `tagName: v__VERSION__`, `releaseName: "YouTube Music Downloader v__VERSION__"`, `releaseDraft: true`, `args: --target universal-apple-darwin`

    **Job 2: build-windows (per D-01, D-02, D-03, D-04)**
    - `runs-on: windows-latest`
    - Steps:
      1. `actions/checkout@v4`
      2. `actions/setup-node@v4` with `node-version: lts/*` and `cache: npm`
      3. `dtolnay/rust-toolchain@stable`
      4. `swatinem/rust-cache@v2`
      5. `npm ci`
      6. `bash scripts/download-sidecars.sh` with `shell: bash` (name: "Download sidecars (Windows)")
      7. `tauri-apps/tauri-action@v0` with:
         - env: `GITHUB_TOKEN`, `TAURI_SIGNING_PRIVATE_KEY`, `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` (from `secrets`). Note: The EV cert signing is handled by `signCommand` in tauri.conf.json which reads `%CERT_THUMBPRINT%` from the environment -- add `CERT_THUMBPRINT: ${{ secrets.CERT_THUMBPRINT }}` to env.
         - with: `tagName: v__VERSION__`, `releaseName: "YouTube Music Downloader v__VERSION__"`, `releaseDraft: true`

    **Job 3: build-linux (per D-12, D-13, D-14, D-17)**
    - `runs-on: ubuntu-22.04` (NOT ubuntu-latest -- pin to 22.04 for glibc compatibility per research Pitfall 4)
    - Steps:
      1. `actions/checkout@v4`
      2. `actions/setup-node@v4` with `node-version: lts/*` and `cache: npm`
      3. `dtolnay/rust-toolchain@stable`
      4. `swatinem/rust-cache@v2`
      5. Install Linux system dependencies:
         ```yaml
         - name: Install Linux dependencies
           run: |
             sudo apt-get update
             sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev \
               librsvg2-dev patchelf libfuse2 file
         ```
      6. `npm ci`
      7. `bash scripts/download-sidecars.sh` (name: "Download sidecars (Linux)")
      8. `tauri-apps/tauri-action@v0` with:
         - env: `GITHUB_TOKEN`, `TAURI_SIGNING_PRIVATE_KEY`, `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` (from `secrets`)
         - with: `tagName: v__VERSION__`, `releaseName: "YouTube Music Downloader v__VERSION__"`, `releaseDraft: true`

    **Important details:**
    - Each job is independent (no `needs:` between them) -- they run in parallel per D-16
    - All three must succeed independently; if any fails, the release draft is incomplete per D-17
    - `releaseDraft: true` means the release is created as a draft -- user manually publishes after verifying artifacts
    - Do NOT add `fail-fast` since these are separate jobs, not a matrix
    - The `tagName` and `releaseName` use `__VERSION__` which tauri-action replaces with the Cargo.toml version
